How to make Subclipse (0.9.24) use JavaSVN library

Though JavaSVN is not yet fully integrated into Subclipse as an extension, you may already try it with Subclipse. Perform the following easy steps to make Subclipse and JavaSVN work together:

  • Install Subclipse (version 0.9.24). You will need Eclipse 3.0 or newer.
  • Download latest version of JavaSVN library. You will need javasvn.jar file from the library distribution.
  • Put javasvn.jar and jsch-0.1.18.jar files into ECLIPSE_HOME/plugins/org.tigris.subversion.subclipse.core_0.9.24/ directory.
  • Modify plugin.xml file in the same directory the following way: 
    <!--
      <library name="svn-javahl.jar">
         <export name="*"/>
      </library>
-->
      <library name="javasvn.jar">
         <export name="*"/>
      </library>
      <library name="jsch-0.1.18.jar">
         <export name="*"/>
      </library>

    Above modification will make Sublipse use JavaSVN version of JavaHL instead of native one. Jsch library is required to enable svn+ssh support. You may also download already modified plugin.xml from here.

  • If you would like not to install svn libraries at all, you may also use patched svnClientAdater.jar that doesn't check for existense of the native svn libraries. You may download it at http://tmate.org/svn/svnClientAdapter.jar.
    Download and put this file into ECLIPSE_HOME/plugins/org.tigris.subversion.subclipse.core_0.9.24/ directory in place of existing svnClientAdapter.jar file.
  • To make Eclipse accept modifications, delete ECLIPSE_HOME/configuration directory and restart Eclipse.
  • When new version of JavaSVN will become available, just replace older version javasvn.jar file with the newer one.

If you have any questions regarding JavaSVN, would like to report a bug or contribute a patch, please write to support@tmatesoft.com
